# Project Summary
The OC Pipeline backend project is an advanced legal practice management system built to streamline workflows in construction project management, particularly for submittal tracking and closeout documentation. Leveraging intelligent agents, the system offers a comprehensive API that supports functionalities such as client management, document processing, billing, and task tracking, all aimed at enhancing efficiency in legal and construction operations.

# Project Module Description
The project consists of multiple functional modules, including:
1. **Authentication**: User registration, login, and JWT management.
2. **Admin**: User management, roles, and audit logs.
3. **Client Management**: Management of client profiles and relationships.
4. **Matter Management**: Tracking cases and workflows.
5. **Document Management**: Storage and versioning of documents.
6. **Billing**: Invoicing and payment processing.
7. **Time Tracking**: Managing billable hours and time entries.
8. **Calendar**: Scheduling events and appointments.
9. **Tasks**: Task creation and management.
10. **Communications**: Messaging and notifications.
11. **Reporting**: Analytics and report generation.
12. **Integrations**: Connecting with third-party services.
13. **Compliance**: Tracking regulatory compliance.
14. **Conflicts**: Managing conflict of interest checks.
15. **Research**: Tools for legal research.
16. **Workflow**: Automated workflow management.
17. **Intelligent Submittal & Construction Documentation Module (ISDC)**: Focused on managing submittals and closeout documentation with AI integration for efficiency.

# File Description Inventory
- **src/**: Contains the main application code including configuration, middleware, routes, controllers, and services.
- **database/**: Holds SQL migration files and documentation related to the database schema.
- **docs/**: Includes deployment guides and API documentation.
- **tests/**: Placeholder for future test suites.
- **README.md**: Project overview and setup instructions.
- **.gitignore**: Files and directories to be ignored by Git.
- **render.yaml**: Configuration file for deployment on Render.

# Technology Stack
- **Runtime**: Node.js 20 with ES Modules
- **Framework**: Express.js
- **Database**: PostgreSQL 15 (via Supabase)
- **Authentication**: Supabase Auth (JWT)
- **ORM**: Direct SQL with pg driver
- **Logging**: Winston
- **Validation**: Joi
- **Testing**: Jest (planned)

# Usage
To set up the project, follow these steps:
1. Clone the repository.
2. Install dependencies using your package manager.
3. Copy the environment template and configure your `.env` file with Supabase credentials.
4. Run database migrations in the Supabase SQL Editor.
5. Start the application in development mode.
